# M.V.R.ESPRINT1 Operational Manual

Version: 0.1.0  
Last updated: April 7, 2026

## Purpose

This manual defines how to run, verify, and troubleshoot the currently implemented M.V.R.ESPRINT1 workflows.

## Intended use

- deterministic pilot demonstrations
- attestation chain validation
- SCED-style CSV chain verification
- local evaluation dashboard execution

## Prerequisites

## Platform

- Linux recommended
- Verified path: Ubuntu 24.04 on WSL2

## Toolchain

- `rustc 1.94.1`
- `cargo 1.94.1`

## Native packages

```bash
sudo apt-get update
sudo apt-get install -y build-essential pkg-config libssl-dev
```

## Workspace verification

```bash
cargo check --message-format short
```

## Installation

```bash
git clone https://github.com/obienova/M.V.R.ESPRINT1.git
cd M.V.R.ESPRINT1
cargo build --release
```

## Optional TPM build

```bash
cargo build --release --features tpm
```

## Runtime configuration

## Environment variable

- `SIGNER_MODE`
  - `simulation` (default evaluation path)
  - `tpm` (optional feature path)

Example:

```bash
export SIGNER_MODE=simulation
```

## Operations

## 1) Validate deterministic SCED chain

```bash
cargo run --bin sced_chain -- verify artifacts/sample_sced.csv
```

Expected CLI contract:

- `Usage: cargo run --bin sced_chain -- verify <input.csv> [expected_hash]`

## 2) Generate and verify sample attestations

```bash
cargo run --bin pilot_demo
cargo run --bin verifier pilot_attestation_log.json
```

## 3) Run deterministic demo scenarios

```bash
cargo run --bin demo -- normal
cargo run --bin demo -- all
```

Supported scenarios:

- `normal`, `reserve`, `capacity`, `network`, `collapse`, `all`

## 4) Start dashboard

```bash
cargo run --bin dashboard
```

Dashboard endpoints:

- `http://127.0.0.1:3000/`
- `http://127.0.0.1:3000/health`
- This is a long-running process; stop it with `Ctrl+C` when finished

## Monitoring and outputs

Primary artifacts and signals:

- `pilot_attestation_log.json` generated by `pilot_demo`
- verifier pass/fail output in terminal
- SCED chain report output in terminal (including final hash and status)
- dashboard health endpoint response

## Routine validation checklist

1. `cargo check --message-format short`
2. `cargo run --bin sced_chain -- verify artifacts/sample_sced.csv`
3. `cargo run --bin pilot_demo`
4. `cargo run --bin verifier pilot_attestation_log.json`

## Troubleshooting

## Build fails

- confirm Rust and Cargo versions
- verify OpenSSL development libraries are installed
- rerun `cargo check --message-format short`

## Verifier fails

- confirm input JSON is valid and complete
- confirm chain continuity (`prev_hash` linkage)
- re-generate records with `pilot_demo`

## SCED validation fails

- check CSV schema and required fields
- check boolean/numeric parse validity
- check for duplicate primary keys in input

## Dashboard not reachable

- verify process is running
- verify `127.0.0.1:3000` is not already in use
- restart dashboard process

## WSL recovery (if environment hangs)

```powershell
wsl --shutdown
wsl -d Ubuntu-24.04
```
